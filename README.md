# FastAPI Logging Service

Small FastAPI service for creating and retrieving log entries during local testing.

## Features

- `GET /health` health check endpoint
- `POST /logs` to store a log entry
- `GET /logs` to list all stored entries
- `GET /logs/{log_id}` to fetch a single entry
- Header-based API key protection on log endpoints
- In-memory storage suited for development only
- Swagger UI and ReDoc generated by FastAPI

## Run Locally

Start the server with Uvicorn:

```bash
uvicorn api.main:app --reload
```

Useful URLs once the server is running:

- Swagger UI: `http://127.0.0.1:8000/docs`
- ReDoc: `http://127.0.0.1:8000/redoc`
- Health check: `http://127.0.0.1:8000/health`

## Authentication

The log endpoints require an `x-api-key` header.

Current demo key:

```text
mysecretkey
```

Example header:

```text
x-api-key: mysecretkey
```

`GET /health` does not require authentication.

## API Endpoints

### `GET /health`

Returns:

```json
{"status": "ok"}
```

### `POST /logs`

Creates a log entry.

Request body:

```json
{
  "level": "INFO",
  "message": "Application started"
}
```

Allowed `level` values:

- `DEBUG`
- `INFO`
- `WARNING`
- `ERROR`
- `CRITICAL`

Example response:

```json
{
  "id": 1,
  "message": "log stored",
  "data": {
    "level": "INFO",
    "message": "Application started"
  }
}
```

### `GET /logs`

Returns all stored logs as a JSON array.

### `GET /logs/{log_id}`

Returns a single log entry by numeric ID.

If the log does not exist, the API returns:

```json
{"detail": "Log not found"}
```

## cURL Examples

Create a log entry:

```bash
curl -X POST "http://127.0.0.1:8000/logs" \
  -H "Content-Type: application/json" \
  -H "x-api-key: mysecretkey" \
  -d "{\"level\":\"INFO\",\"message\":\"Application started successfully\"}"
```

List all logs:

```bash
curl "http://127.0.0.1:8000/logs" \
  -H "x-api-key: mysecretkey"
```

Fetch one log:

```bash
curl "http://127.0.0.1:8000/logs/1" \
  -H "x-api-key: mysecretkey"
```

## Notes

- Log data is stored in memory and is lost when the process restarts.
- The API key is hardcoded and suitable only for local testing or learning.
